Welcome to HeartBleed!This is my first paper.I will introduce how to install Apache with SSL on Linux and test the heartbleed with python.

1.Configure Environment

1.1. Download Apache And Openssl

Download Apache from Apache and Openssl from Openssl with 1.0.1f(this has heartbleed).

1.2. Install Openssl and Apache

1.2.1. First Openssl

If your computer has installed openssl ,you just don’t need to care about that.

This openssl will be installed in “/usr/local/openssl”

1.2.2. Then Apache

Apache will be installed together with openssl.

Maybe there will be a problem:”error140770fcssl-routinesssl23-get-server-hellounknown-protocol”,you can use “export LDFLAGS=-ldl” to solve it.

Then install apache

1.3. Enable SSL in httpd.conf

Apache configuration file httpd.conf is located under “/usr/local/apache/conf”

Uncomment the httpd-ssl.conf Include line in the “/usr/local/apache/conf/httpd.conf” file

Delete “#” before “#Include conf/extra/httpd-ssl.conf”

View the httpd-ssl.conf to review all the default SSL configurations. In most cases, you don’t need to modify anything in this file.

The SSL certificate and key are required before we start the Apache. The server.crt and server.key file mentioned in the httpd-ssl.conf need to be created before we move forward.

Confirm the position.

1.4. Create server.crt and server.key file

1.4.1. First, Generate the server.key using openssl.

The above command will ask for the password. Make sure to remember this password. You need this while starting your Apache later.

1.4.2. Next, generate a certificate request file (server.csr) using the above server.key file.

1.4.3. Finally, generate a self signed ssl certificate (server.crt).

1.5. Start the apache and verify SSL

Start the Apache as shown below.

This will prompt you to enter the password for your private key.If not ,please use command “/usr/local/apache/bin/apachectl restart”

By default Apache SSL runs on 443 port. Open a web browser and verify that you can access your Apache using https://your-ip-address
